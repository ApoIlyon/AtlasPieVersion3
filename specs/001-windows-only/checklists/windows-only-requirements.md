# Requirements Checklist: Windows-only AtlasPie Cleanup

## Requirement Completeness
- CHK001: Описаны ли все типы Linux/macOS артефактов (файлы, каталоги, ресурсы) и соответствующие действия по их удалению, чтобы избежать «немых» объектов в репозитории? [Completeness]
- CHK002: Зафиксированы ли обязательные изменения для всех групп конфигураций (Cargo, tauri.conf.json5, package.json, GitHub Actions) так, чтобы ни один конфиг не остался без Windows-only инструкций? [Completeness]
- CHK003: Указаны ли обязательные обновления для документации/тестов (README, quickstart, Playwright, Vitest) с перечислением всех артефактов, которые нужно переписать под Windows? [Completeness]

## Requirement Clarity
- CHK004: Содержат ли требования по очистке кода явные указания, какие файлы/модули должны остаться после удаления `#[cfg(target_os)]` веток? [Clarity]
- CHK005: Уточнены ли критерии для сохранения «legacy» упоминаний (например, где можно оставить ссылки на Linux/macOS и как их пометить)? [Clarity]

## Requirement Consistency
- CHK006: Согласованы ли требования фронтенда и backend так, чтобы нигде не упоминались отсутствующие Tauri-команды или UI-блоки после удаления Linux/macOS функциональности? [Consistency]
- CHK007: Совпадают ли ожидания из Success Criteria (SC-001…SC-005) с задачами план/таск-листа (например, поиск ключевых слов, успешные пайплайны)? [Consistency]

## Acceptance Criteria Quality
- CHK008: Предусмотрены ли измеримые проверки для каждого пользовательского сценария (поиск ключевых слов, `cargo/pnpm` прогон, обновлённые документы), чтобы легко подтвердить выполнение? [Acceptance]

## Scenario & Edge Case Coverage
- CHK009: Описаны ли правила обработки исторических/архивных ссылок (changelog, legacy docs) и подтверждено ли, что автоматические проверки их исключают из поиска? [Edge Case]
- CHK010: Указаны ли действия при обнаружении зависимостей, которые формально кроссплатформенные, но используются только ради Windows? [Scenario Coverage]

## Non-Functional Requirements
- CHK011: Включены ли требования к производительности пайплайна (время выполнения `cargo/pnpm/test`) и отражено ли это в планируемых проверках? [Non-Functional]
- CHK012: Заданы ли ограничения на инструменты автоматизации (PowerShell, скрипты проверки), чтобы обеспечить воспроизводимость на чистой Windows машине? [Non-Functional]

## Dependencies & Assumptions
- CHK013: Перечислены ли критичные предположения (например, наличие Windows build tools, отсутствие сторонних Linux runners) и способы их подтверждения? [Dependencies]

## Ambiguities & Conflicts
- CHK014: Уточнено ли, требуется ли сохранять какие-либо кроссплатформенные зависимости «до лучших времён», и описаны ли критерии, когда их можно оставить? [Ambiguity]
